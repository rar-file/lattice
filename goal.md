The non-negotiable principle
The markdown vault on disk is the only source of truth. SQLite, embeddings, FTS indexes, every byte of derived state — all of it must be rebuildable from the vault with one command: lattice rebuild. Every architectural decision flows from this.

If a feature needs Lattice-specific metadata that doesn't round-trip through plain markdown, stop and reconsider. This is what makes "point at your Obsidian folder and go" actually work — the migration story isn't a feature, there's nothing to migrate, the vault IS the storage.


Scope
Ships in v1:

Vault watcher and incremental indexer (markdown, frontmatter, wikilinks, tags, attachments)
Markdown editor on web and desktop with Obsidian-compatible wikilink parsing
Global capture inbox — hotkey, small floating window, file lands in Inbox/
Hybrid search (FTS plus vector, fused with reciprocal rank fusion)
Ambient links — right-rail panel showing semantically related notes as you write
MCP server exposing the vault to Claude Desktop, Cursor, Cline, Zed, etc.
Obsidian-compatible vault format (read any existing Obsidian folder unchanged — a side effect of speaking plain markdown, not a separate import flow)
Cut from v1, no exceptions:

Weekly synthesis (depends on search being rock-solid first)
Recall summaries
Mobile
Graph view (screenshots well, doesn't move retention)
Plugin system
Multi-vault
Collaboration or sync
A "chat with your notes" tab as the primary UI
If a feature isn't on the first list, defer it. The MVP is small on purpose.


Architecture
Four surfaces, one seam, three core services, one storage interface, two backends, one vault. Top to bottom:

Surfaces: CLI Web Desktop (Tauri) MCP server
↓
Seam: FastAPI
↓
Core: Indexer Hybrid search Embedder
↓
Abstraction: Storage interface (Repo protocol)
↓ ↓
Backends: SQLite + sqlite-vec Postgres + pgvector
↓ ↓
Vault: Markdown filesystem

Every surface, including the Tauri webview, talks to FastAPI over HTTP. No special-case "desktop accesses storage directly" path. The desktop bundles the FastAPI binary as a sidecar bound to 127.0.0.1 with a per-launch bearer token in the Authorization header. Local sidecars must require auth. Hard requirement, not a v2 thing.

The Repo protocol is the load-bearing abstraction. SQLiteRepo and PostgresRepo implement the same interface. Core services never import sqlite3 or asyncpg directly — the day one does is the day Postgres parity becomes a rewrite.

Embedder and Chat protocols follow the same pattern. Embedder implementations: OpenAIEmbedder, VoyageEmbedder, OllamaEmbedder. Chat implementations: AnthropicChat, OpenAIChat, OllamaChat. Configured via env, swappable at runtime.


Design system — read this twice
The single most likely failure mode is producing a generic AI SaaS UI: purple-to-pink gradients, sparkle icons on every "AI" feature, rainbow accent colours per section, emoji as decoration, "Ask Lattice" buttons, oversized rounded cards with drop shadows, three different gradient backgrounds. Lattice looks nothing like that.

Visual references, in priority order: Linear (chrome, density, command palette), iA Writer (the editor surface), Stripe Docs (typography and tables), Vercel dashboard (restraint, monochrome), Cron / Notion Calendar (calm pacing of secondary chrome). Study these before writing CSS. The goal is a tool that looks designed by someone with taste, not generated by an LLM with default Tailwind.

Typography
Two families maximum. Inter or Geist Sans for chrome, navigation, UI labels. Source Serif or iA Writer Quattro optionally for the editor body (the reading surface earns the serif). JetBrains Mono or Berkeley Mono for metadata, frontmatter, tags, code, dates.

Type sizes on a strict scale: 12 / 13 / 14 / 16 / 20 / 28. Two weights only: 400 regular, 500 medium. Never 600 or 700 — they read heavy and modern UIs use weight contrast, not weight extremes.

Colour
Monochrome base. True black text in light mode, off-white text in dark mode. One accent colour used exclusively for interactive states: focus rings, active selection, the indicator that AI is currently streaming. The accent is not a brand colour painted onto buttons. Pick a desaturated blue or muted amber — not saturated purple, not pink, not anything gradient.

Semantic colour (red for destructive, green for success) appears only at the moment it's needed, never as ambient decoration.

Anti-patterns, banned
Gradients of any kind
Glass morphism, frosted blur, backdrop filters
Drop shadows beyond a single 1px solid rgba(0,0,0,0.06) hairline on elevated surfaces
Glow, neon, animated background mesh
Sparkle icons, magic wand icons, anything labelled "✨ AI"
Emoji as UI elements (user-typed emoji in note content is fine; emoji in chrome is not)
More than one accent colour
Stock illustrations with floating UI screenshots
Borders, radii, spacing
Borders 0.5px or 1px, never thicker. Radius 6px default, 8px for cards. Never pill except for deliberate badges. Spacing on a 4px grid: 4, 8, 12, 16, 24, 32, 48, 64. No values outside the scale.

Icons
A single outline icon set (Lucide or Tabler outline). 16px or 20px only. Icons inherit text colour. No filled variants.

AI features get a quiet visual identifier: a thin left border in the accent colour, or nothing at all. Never a sparkle.

Layout
The editor is the application. Sidebars are subordinate, collapsible, low-contrast.

Default desktop layout: 240px left rail (file tree, tags), centre column (editor, max 720px measure), 320px right rail (backlinks above the fold, related notes below, AI suggestions at the bottom). On narrower widths the rails collapse to icons, then disappear. Cmd-K command palette does everything else.

Motion
150ms ease-out for most transitions, 100ms for hover states. No bounce, no spring, no "playful" easing. Page transitions: opacity only, no slide. Skeleton loaders, not spinners. If something takes longer than 400ms, show a thin top progress bar (YouTube / GitHub style), not a spinner.

AI surface treatments
This is where most apps embarrass themselves. Lattice rules:

AI-generated suggestions live in the margin (right rail), never inline modal popups
Streaming text uses a thin caret cursor — no glowing borders, no shimmering placeholders
"AI is thinking" is a single 1px line pulsing at the top of the relevant panel, nothing else
Never use the word "magic." Features are named what they are: "Related notes," "Synthesize," "Capture"
AI errors are quiet — a small grey footer "Couldn't reach provider. Try again." — not a red banner

Surface-by-surface specs
Web app — Next.js App Router
Editor uses CodeMirror 6, not Tiptap. CM6's source-of-truth model maps cleanly onto markdown-on-disk; Tiptap's ProseMirror schema fights you when the on-disk format is markdown. Write CM6 extensions for wikilink completion ([[ opens a fuzzy picker), inline tag highlighting, frontmatter folding, image preview in margin. Live preview is single-source — markdown renders in place, source visible on focus, no separate preview pane.

Desktop — Tauri v2
Wraps the web app in a native window. Bundles FastAPI as a sidecar via PyInstaller (or Nuitka if startup time matters). Sidecar binds 127.0.0.1 with a per-launch token. Desktop adds: global capture hotkey (default Cmd-Shift-N / Ctrl-Shift-N), system tray icon for "Capture" and "Show vault," OS-level file watching that's more reliable than cross-platform watchdog alone.

CLI — Click
Commands: lattice index <path>, lattice search <query>, lattice rebuild, lattice capture (stdin to inbox), lattice serve (start API), lattice mcp (start MCP server). The CLI talks to the API, not the storage layer. Tab completion via Click built-ins.

MCP server — the moat
Six tools, no more for v1:

search(query, limit=10, mode="hybrid"|"semantic"|"keyword") — results with note path, heading path, snippet, score
read_note(path) — frontmatter plus body
write_note(path, content, mode="overwrite"|"append") — confirms write, returns new mtime
append_to_daily(content, date="today") — appends to today's daily note
list_backlinks(path) — notes linking to this one
recent(limit=20, since=null) — recently modified notes
Document it well enough that someone configuring Claude Desktop with Lattice never reads the source.

Capture inbox
Global hotkey opens a 480px floating window. Single textarea, optional tag chip row, Cmd-Enter saves and dismisses. Hotkey to focused input must be under 200ms. Saves to Inbox/YYYY-MM-DD.md with a timestamp, appending if the file exists. No "select destination" dialog — capture is for friction-free thoughts; organising happens later.


Tech choices, baked in
Python 3.12, async everywhere. FastAPI, asyncpg, aiosqlite, httpx
Indexer: watchdog for file events, debounced 250ms. Each note gets an xxhash content hash; chunks get the same. Re-embed only on chunk hash delta
Chunking: heading-aware semantic chunking. Each chunk carries note_id, heading_path (e.g. Postgres > Indexes > BRIN), position, content_hash. Target ~512 tokens per chunk with overlap on heading boundaries
Search ranking: reciprocal rank fusion (k=60) over FTS5 and vector cosine similarity results. Tune later; RRF is fine for v1
Embeddings: bge-small-en-v1.5 via sentence-transformers as local default. voyage-3 for hosted tier when configured
Frontend: Next.js App Router, CodeMirror 6, Tailwind for utility-only chrome (NOT for component design — components are hand-built), @tanstack/react-query for API state, no global state library
Desktop: Tauri v2, FastAPI sidecar via PyInstaller
Testing: pytest with a fixture vault containing ten real-world markdown patterns (Obsidian-style frontmatter, callouts, Dataview inline fields, nested tags, aliases, attachments, embedded notes, emoji filenames, malformed wikilinks, mixed line endings)

Build order
Week 1 — Foundations. Repo protocol, SQLiteRepo, vault scanner, file watcher, Embedder protocol with one provider wired up. End state: lattice index ~/vault && lattice search "x" returns FTS results.

Week 2 — API plus search. FastAPI exposing vault operations, sqlite-vec wired in, hybrid search with RRF, CLI talking to the API. End state: semantic search returns sensible results on a real Obsidian vault.

Week 3 — Web app. Next.js, CodeMirror 6 editor, file tree, search bar with live results, ambient-links right rail, capture modal. Busiest week — budget extra time for the editor.

Week 4 — MCP plus Tauri plus polish. MCP server with six tools, Tauri shell bundling the FastAPI sidecar, first-run flow (pick or create a vault folder), installer, demo path.

Do not skip week 1 to start the UI early. The indexer being correct is the difference between a working product and a toy.


Acceptance: the first-session demo
The MVP is done when this works without scripting or hand-holding, on a vault the user created today:

User installs Lattice; total time from download to a focused capture window is under 60 seconds. The first-run flow asks one question — "where should your vault live?" — and that's it
They hit the global capture hotkey, type a thought, Cmd-Enter — file lands in Inbox/YYYY-MM-DD.md as plain markdown, no further prompts
They write a longer note. As they type past the first few paragraphs, the right rail starts surfacing semantically related notes from elsewhere in the vault, even when that vault only has a dozen notes in it. Ambient links must feel useful at small scale, not just at scale
They configure Claude Desktop with Lattice's MCP server — three lines of JSON, copy-pasted from a "Connect Claude" panel in Lattice's settings. Claude can now read and write to the vault
They ask Claude: "Pull together what I've been thinking about [topic] this week and draft a summary" — Claude searches the vault, reads the relevant notes, drafts a summary, saves it back as a new note in Drafts/, all through MCP. The user watches the new file appear in Lattice's file tree in real time
The user never opened a settings panel beyond step 4, never picked an embedding model, never installed a plugin, never thought about chunking or indexing
The promise being tested: AI features work out of the box on a vault that didn't exist yesterday, and grow more valuable as the vault grows. If that lands, v1 ships. If it doesn't, additional features won't save it.


What will get rejected on review
Any visual element using a gradient
Sparkle icons, magic wand icons, or any "✨ AI" labelling
Emoji as UI elements (in user content fine, in chrome not)
More than one accent colour
A core service importing sqlite3 or asyncpg directly
A capture flow that takes more than 200ms to reach focused input
A "chat with your notes" tab anywhere in the UI
An onboarding wizard
A settings page longer than one screen
Anything that requires the user to organise notes before they get value
A local sidecar that accepts unauthenticated requests
When in doubt, choose the more restrained option. Lattice's competitive position is being the calm, fast, well-designed alternative in a category full of cluttered apps and shimmering AI demos.
